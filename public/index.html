    Why obfuscate your Lua code
    
        
    Protect the work you put into your scripts.
    
      
    
      
    
        
    
          
          
    Protect your logic
    
          
    Stop others from copying your scripts and passing them off as their own.
    
        
    
        
    
          
          
    Harder to reverse
    
          
    Variable renaming, control-flow changes, and string encoding make manual reading painful.
    
        
    
        
    
          
          
    Guard against tampering
    
          
    Makes it harder for others to quietly patch in unwanted behavior before redistributing your script.
    
        
    
        
    
          
          
    Free to automate
    
          
    Use the API to fold obfuscation into your own build or release pipeline.
    
        
    
      
    
    
    

    
    
    
      
    
        
    API reference
    
        
    Generate a key, then call the endpoint from your own tools.
    
      
    

      
    
        
        
    
          
          
        
    
        
    Demo keys reset when the server restarts. Store your real key somewhere safe once you go live.
    
      
    

      
    
        
    POST/api/v1/obfuscate
    
        
    Obfuscates a Lua source string.
    
        
    Headers
    
        
    x-api-key: YOUR_KEY
Content-Type: application/json
    
        
    Body
    
        
    {
  "code": "print('hello world')",
  "preset": "Medium"
}
    
        
    Response
    
        
    {
  "ok": true,
  "output": "-- obfuscated lua here"
}
    
      
    

      
    
        
    GET/api/presets
    
        
    Lists the available obfuscation presets.
    
      
    

      
    
        
    Example (curl)
    
        
    curl -X POST https://your-app.onrender.com/api/v1/obfuscate \
  -H "x-api-key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"code":"print(1)","preset":"Strong"}'
    
      
    

      
    
        
    Rate limits
    
        
    12 requests per minute per IP address on obfuscation endpoints. Contact the maintainer for higher limits.
    
      
    
    
    

    
    
    
      
    
        
    Privacy policy
    
        
    Short version: your code passes through, it doesn't stay.
    
      
    
      
    
        
    What we receive
    
        
    The Lua code you paste or upload, and the preset you choose, are sent to the server only to run the obfuscation.
    
        
    What we store
    
        
    Nothing server-side. Temporary files are deleted immediately after each job finishes. Your run history is saved only in your own browser, not on our servers.
    
        
    API keys
    
        
    Keys are held in memory for the current server session and are cleared on restart. Don't submit code you can't afford to have pass through a third-party server.
    
        
    Logging
    
        
    Standard request logs (timestamp, IP, status code) may be kept briefly for abuse prevention, never the code content itself.
    
        
    Contact
    
        
    Questions about this policy can go to the maintainer directly.
